## 1. 개요

### 1.1 개선 배경

기존 시스템의 주요 약점:
- **Breakout**: 6개 조건 모두 만족 필요 → 진입 기회 상실
- **Mean Reversion**: 반등 확인 없이 진입 → "떨어지는 칼날" 위험
- **리스크 관리**: 고정 포지션 사이징 → 시장 상황 미반영
- **v2.0 문제점**: 진입 조건이 너무 느슨 → 가짜 돌파 다수 발생

### 1.2 개선 목표

| 영역 | 목표 |
|------|------|
| 진입 로직 | **엄격한 필터링** + 가짜 신호 제거 |
| 청산 로직 | 수익 보호 강화 + 넓은 손절폭 |
| 리스크 관리 | 동적 조정 + 자본 보호 강화 |

### 1.3 버전별 변경 요약

```
v2.0 (Phase 1 + 2)
├── 4단계 트레일링 스탑 (Breakout)
├── 점수 기반 진입 (Breakout)
├── 양봉 확인 진입 (Mean Reversion)
├── 부분 익절 시스템
└── 동적 포지션 사이징

v2.2 (가짜 신호 필터링 강화)
├── 3봉 연속 돌파 확인 (2봉 → 3봉)
├── EMA 정배열 필수 (EMA21 > EMA50 > EMA200)
├── ADX > 25 필수 (강한 추세만)
├── RSI 과열 필터 (45-70)
├── 손절폭 확대 (2.0 → 3.0 ATR)
├── Mean Reversion 단순화 + 폭락장 필터
└── 트레일링 간격 확대

v3.0 (시장 레짐 기반 시스템) ⭐ 최신
├── 시장 레짐 분류 시스템 (TREND_UP/DOWN, RANGING, WEAK_TREND)
├── 레짐 게이트: 전략별 진입 허용 레짐 설정
├── Breakout 조건 완화 (3봉→2봉, RSI 55→45, 점수 5→2)
├── Mean Reversion 2단계 부분 익절 시스템
├── Momentum Strategy 신규 추가 (RSI 상향 돌파 전략)
├── 포트폴리오 레벨 TREND_DOWN 포지션 제한
├── ATR 기반 슬리피지 모델 추가
├── Walk-Forward 백테스트 (랜덤→체계적 검증)
└── 전략 가중치 조정 (Breakout 0.4, Mean Rev 0.3, Momentum 0.3)
```

## 2. v3.0 개선 사항 (시장 레짐 시스템)

### 2.1 시장 레짐 분류 시스템

v3.0의 핵심 혁신은 **시장 상태를 4가지 레짐으로 분류**하고 전략별로 진입 허용 레짐을 설정한 것입니다.

#### 2.1.1 레짐 분류 기준

| 레짐 | 조건 | 의미 |
|------|------|------|
| **TREND_UP** | EMA21 > EMA50 > EMA200<br>ADX > 25<br>종가 > EMA50 | 강한 상승 추세 |
| **TREND_DOWN** | EMA21 < EMA50 < EMA200<br>ADX > 25<br>종가 < EMA50 | 강한 하락 추세 |
| **RANGING** | ADX < 20 | 횡보장 (방향성 없음) |
| **WEAK_TREND** | 나머지 (ADX 20~25) | 약한 추세 |

#### 2.1.2 레짐 강도

- `regime_strength` = ADX 값 (0~100)
- ADX가 높을수록 추세가 강함

#### 2.1.3 설정 파라미터

```python
REGIME_ADX_TREND = 25  # 추세 인정 임계값
REGIME_ADX_RANGE = 20  # 횡보 인정 임계값
```

### 2.2 전략별 레짐 게이트

각 전략은 특정 시장 상태에서만 진입합니다.

| 전략 | 허용 레짐 | 차단 레짐 | 논리 |
|------|-----------|-----------|------|
| **Breakout** | TREND_UP | TREND_DOWN, RANGING, WEAK_TREND | 강한 상승 추세만 돌파 추종 |
| **Mean Reversion** | RANGING, WEAK_TREND | TREND_UP, TREND_DOWN | 횡보장에서만 역추세 |
| **Momentum** | TREND_UP, WEAK_TREND | TREND_DOWN, RANGING | 상승/약추세에서 모멘텀 |

### 2.3 포트폴리오 레벨 레짐 관리

**TREND_DOWN 시 포지션 제한:**
- 이미 포지션 보유 → 신규 진입 차단
- 포지션 없음 → 최대 1개만 진입 허용

**목적:** 하락장에서 자본 보호

### 2.4 Breakout Strategy v3.0 변경점

| 항목 | v2.2 | v3.0 | 변경 사유 |
|------|------|------|-----------|
| **레짐 필터** | - | **TREND_UP만 진입** | 레짐 게이트 도입 |
| 저항선 돌파 | 3봉 연속 | **2봉 연속** | 레짐 필터로 신뢰도 확보, 완화 |
| ADX 조건 | > 25 필수 | **제거** | 레짐 필터가 대체 |
| RSI 범위 | 45-70 | **45-70 유지** | - |
| 최소 점수 | 5점 | **2점** | 진입 조건 완화 |
| 거래량 필터 | 1.5x 필수 | **점수 보너스** | 필수→선택 완화 |
| 트레일링 stage1 | 3.5 ATR | **4.0 ATR** | 여유 확대 |
| 트레일링 stage3 | 10.0 ATR | **12.0 ATR** | 대형 수익 보호 강화 |

**핵심 철학:** 레짐 필터가 시장 상태를 검증하므로 개별 조건 완화 가능

### 2.5 Mean Reversion Strategy v3.0 변경점

| 항목 | v2.2 | v3.0 | 변경 사유 |
|------|------|------|-----------|
| **레짐 필터** | - | **RANGING 또는 WEAK_TREND** | 횡보장만 진입 |
| RSI | < 35 | **< 30** | 더 강한 과매도 확인 |
| 폭락장 필터 | EMA200 × 0.93 | **EMA200 × 0.97** | 3% 이내로 강화 |
| 거래량 필터 | - | **>= 1.5배 평균** | 신뢰도 확보 |
| 청산 방식 | BB 중심선 전량 익절 | **2단계 부분 익절** | 수익 극대화 |

#### 2.5.1 2단계 부분 익절 시스템

```
1단계: BB 중심선 도달 → 50% 익절 (EXIT_TP_MEAN_PARTIAL)
2단계: BB 상단 도달 → 나머지 익절 (EXIT_TP_MEAN_FULL)
```

#### 2.5.2 부분 익절 후 보호 로직

부분 익절 후 가격 하락 시:
- 손익분기점으로 손절 이동
- 1.5 ATR 하락 시 전량 청산 (EXIT_MEAN_PROTECT)

#### 2.5.3 손절 파라미터

```python
MEAN_REV_SL_ATR_MULT = 2.0  # 손절폭 ATR 배수
MEAN_REV_SL_MIN_PCT = 0.004  # 최소 손절폭 0.4%
```

#### 2.5.4 시간 기반 조기 탈출

| 보유 시간 | 동작 |
|-----------|------|
| 144봉 (12시간) | 손절가를 1.0 ATR로 축소 |
| 216봉 (18시간) | 강제 청산 (EXIT_MEAN_TIME) |

### 2.6 Momentum Strategy (신규)

RSI 상향 돌파를 이용한 모멘텀 전략.

#### 2.6.1 진입 조건

| 조건 | 값 | 설명 |
|------|-----|------|
| 레짐 | TREND_UP 또는 WEAK_TREND | 상승/약추세만 |
| RSI 돌파 | RSI > 55 (전봉 RSI < 55) | 55 상향 돌파 |
| RSI 기울기 | 최근 5캔들 양수 | 상승 모멘텀 확인 |
| EMA 정배열 | EMA21 > EMA50 | 단기 상승 추세 |
| 가격 위치 | 종가 > EMA50 | 중기선 위 |
| 거래량 | >= 1.5배 평균 | 거래량 확인 |
| 양봉 비율 | 최근 5봉 중 3봉 이상 | 강한 상승 |

#### 2.6.2 2단계 부분 익절

```
1단계: RSI 75+ 도달 후 10pt 하락 → 50% 익절 (EXIT_TP_MOM_PARTIAL)
2단계: RSI < 65로 하락 → 나머지 익절 (EXIT_TP_MOM_FULL)
```

#### 2.6.3 손실 보호 청산

RSI 하락 + 손실 상태 → 전량 청산 (EXIT_MOM_LOST)

#### 2.6.4 2단계 트레일링 스탑

| 수익 구간 | 트레일링 간격 |
|-----------|---------------|
| 손익분기 | 2.0 ATR (손익분기로 이동) |
| 3.0 ATR 도달 | 1.5 ATR (타이트 추격) |

#### 2.6.5 손절 파라미터

```python
MOMENTUM_SL_ATR_MULT = 2.5  # 손절폭 ATR 배수
MOMENTUM_SL_MIN_PCT = 0.005  # 최소 손절폭 0.5%
```

### 2.7 전략 가중치 변경

| 전략 | v2.2 | v3.0 | 변화 |
|------|------|------|------|
| Breakout | 0.6 | **0.4** | -0.2 |
| Mean Reversion | 0.4 | **0.3** | -0.1 |
| Momentum | - | **0.3** | 신규 |

**목적:** 3개 전략으로 포트폴리오 다각화

### 2.8 슬리피지 모델 추가

ATR 기반 동적 슬리피지:

```python
slippage = ATR × 5%
slippage = max(0.01%, min(slippage, 0.1%))  # 0.01%~0.1% 범위
```

**적용:**
- 매수가 = 신호가 × (1 + slippage)
- 매도가 = 신호가 × (1 - slippage)

### 2.9 Walk-Forward 백테스트

`multi_period_backtest.py`를 랜덤 기간 테스트에서 체계적 Walk-Forward 분석으로 전환.

#### 2.9.1 주요 개선점

| 항목 | v2.2 (랜덤) | v3.0 (Walk-Forward) |
|------|-------------|---------------------|
| 기간 선택 | 100개 랜덤 기간 | 체계적 순차 윈도우 |
| In/Out-of-Sample | 구분 없음 | 70% IS / 30% OOS |
| 모드 | - | Rolling 또는 Anchored |
| 데이터 스누핑 방지 | 약함 | 강함 (비중첩 테스트) |

#### 2.9.2 설정 파라미터

```python
BACKTEST_WINDOW_SIZE = 5000  # 전체 윈도우 캔들 수
BACKTEST_IN_SAMPLE_PCT = 0.7  # In-sample 비율 (70%)
BACKTEST_STEP_SIZE = 1500  # Rolling 윈도우 스텝 크기
BACKTEST_MODE = 'rolling'  # 'rolling' 또는 'anchored'
```

#### 2.9.3 동작 방식

**Rolling 모드:**
```
Window 1: [0     ~ 5000]   → IS: [0 ~ 3500], OOS: [3500 ~ 5000]
Window 2: [1500 ~ 6500]   → IS: [1500 ~ 5000], OOS: [5000 ~ 6500]
Window 3: [3000 ~ 8000]   → IS: [3000 ~ 6500], OOS: [6500 ~ 8000]
...
```

**Anchored 모드:**
```
Window 1: [0 ~ 5000]      → IS: [0 ~ 3500], OOS: [3500 ~ 5000]
Window 2: [0 ~ 6500]      → IS: [0 ~ 4550], OOS: [4550 ~ 6500]
Window 3: [0 ~ 8000]      → IS: [0 ~ 5600], OOS: [5600 ~ 8000]
...
```

## 6. Momentum Strategy (신규)

### 6.1 전략 개요

v3.0에서 추가된 세 번째 전략으로, **RSI 상향 돌파**를 통한 모멘텀 추종 전략입니다.

**핵심 아이디어:** RSI가 55를 상향 돌파할 때 상승 모멘텀이 시작된다고 판단하고 진입합니다.

### 6.2 진입 조건

| 조건 | 값 | 설명 |
|------|-----|------|
| **레짐 게이트** | TREND_UP 또는 WEAK_TREND | 상승/약추세에서만 진입 |
| **RSI 돌파** | RSI > 55 & 전봉 RSI < 55 | 55 상향 돌파 순간 |
| **RSI 기울기** | 최근 5캔들 양수 | RSI가 상승 중 |
| **EMA 정배열** | EMA21 > EMA50 | 단기 상승 추세 |
| **가격 위치** | 종가 > EMA50 | 중기선 위 |
| **거래량** | >= 1.5배 평균 | 거래량 확인 |
| **양봉 비율** | 최근 5봉 중 3봉 이상 | 강한 상승 |

### 6.3 2단계 부분 익절

```
1단계: RSI 75+ 도달 후 10pt 하락 → 50% 익절 (EXIT_TP_MOM_PARTIAL)
       (예: RSI 80 → 70으로 하락 시)

2단계: RSI < 65로 하락 → 나머지 익절 (EXIT_TP_MOM_FULL)
```

**논리:** RSI가 과열(75+) 후 하락하면 모멘텀 약화 신호

### 6.4 손실 보호 청산

```
조건: RSI 하락 중 & 손실 상태
청산: 전량 청산 (EXIT_MOM_LOST)
```

**목적:** 모멘텀 소실 시 빠른 손절

### 6.5 2단계 트레일링 스탑

| 수익 구간 | 트레일링 간격 | 설명 |
|-----------|---------------|------|
| < 2.0 ATR | 고정 손절가 | 손절가 유지 |
| 2.0 ATR 도달 | 손익분기로 이동 | 손실 방지 |
| 3.0 ATR 도달 | 1.5 ATR 트레일링 | 타이트 추격 |

```
v3.0 수익 흐름:
진입 → 2ATR(손익분기 이동) → 3ATR(1.5ATR 트레일링 시작)
```

## 7. 리스크 관리 시스템 개선

### 7.1 동적 포지션 사이징

#### 변동성 기반 리스크 조정

| ATR/가격 비율 | 시장 상태 | 거래당 리스크 |
|---------------|-----------|---------------|
| < 1% | 저변동성 | **1.5%** (공격적) |
| 1% ~ 2% | 중간 | **1.0%** (기본) |
| > 2% | 고변동성 | **0.7%** (보수적) |

### 7.2 드로우다운 기반 리스크 조정

| 일일 DD | 리스크 배수 | 신규 진입 | 상태 |
|---------|-------------|-----------|------|
| < 2% | 1.0x | 허용 | 정상 |
| 2% ~ 4% | 0.7x | 허용 | 주의 |
| 4% ~ 6% | 0.5x | **제한** | 경고 |
| > 6% | 0.0x | **금지** | **거래 중단** |

### 7.3 동시 포지션 관리

| 상황 | 리스크 조정 |
|------|-------------|
| 첫 번째 포지션 | 정상 리스크 |
| 두 번째 포지션 진입 시 | 각 포지션 리스크 **0.7x** |
| 세 번째 이상 | **진입 불가** |

### 7.4 리스크 계산 흐름

```
기본 리스크 (변동성 기반)
    ↓
× DD 배수 (드로우다운 기반)
    ↓
× 동시 포지션 배수 (0.7x if 2nd position)
    ↓
= 최종 리스크
```
